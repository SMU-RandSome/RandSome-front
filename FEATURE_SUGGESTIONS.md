# Randsome 기능 제안서

> 현재 구현된 서비스를 분석하여, 사용자(회원)와 관리자 양쪽 관점에서 필요하거나 있으면 좋을 기능들을 정리한 문서입니다.

---

## 1. 사용자(회원) 관점

### 1-1. 매칭 경험 향상

#### 매칭 만족도 피드백
- 매칭 결과 열람 후 **별점(1~5) + 한줄 후기** 남기기
- "다시 매칭 받고 싶어요" 재신청 바로가기 버튼
- 왜 필요한가: 서비스 품질 측정 + 알고리즘 개선 데이터 확보

#### 매칭 결과 상세 비교
- 매칭된 상대의 프로필을 **카드 스와이프** 형태로 비교
- MBTI 궁합 점수 표시 (예: ENFP ↔ INTJ → 궁합 85%)
- 왜 필요한가: 여러 명 매칭 시 한눈에 비교하기 어려움

#### 이상형 작성 가이드
- 이상형 입력 시 **예시 문구** 또는 **태그 선택** 제공 (예: #유머있는 #다정한 #운동좋아하는)
- 텍스트만 입력하면 막연해서 퀄리티가 떨어짐
- 왜 필요한가: 이상형 기반 매칭 알고리즘의 입력 품질 향상

#### 매칭 히스토리 타임라인
- 내 모든 매칭 신청/결과를 **타임라인 형태**로 시각화
- 신청일 → 승인일 → 결과 열람일 진행 상태 표시
- 왜 필요한가: 현재는 탭으로만 구분되어 전체 흐름 파악이 어려움

---

### 1-2. 프로필 & 소통

#### 프로필 매력도 게이지
- 프로필 완성도를 **퍼센트 바**로 표시
- 자기소개 길이, 이상형 작성 여부, 인스타그램 연결 등에 따라 점수 부여
- "프로필을 더 채우면 매칭 확률이 올라가요!" 넛지 메시지
- 왜 필요한가: 빈약한 프로필로 매칭 퀄리티 저하 방지

#### 매칭 후 아이스브레이커
- 매칭 결과 확인 시 **대화 시작 추천 멘트** 제공
- MBTI 기반 대화 주제 추천 (예: "ENFP인 OOO님에게는 여행 이야기로 시작해보세요!")
- 왜 필요한가: 인스타 DM을 보내기 어색한 사용자를 위한 UX 배려

#### 공지사항 & FAQ
- 현재 MyPage에 더미 공지만 있음 → **실제 공지사항 API 연동**
- 자주 묻는 질문(FAQ) 섹션 추가
  - "입금했는데 승인이 안 돼요"
  - "매칭 결과는 언제 나오나요?"
  - "환불이 가능한가요?"
- 왜 필요한가: 운영 중 반복 문의 감소

---

### 1-3. UX 개선

#### 진행 상태 실시간 안내
- 후보 등록/매칭 신청 후 **현재 상태 + 예상 소요 시간** 표시
- "관리자가 입금을 확인 중입니다 (평균 10분 소요)"
- 프로그레스 스텝 UI: `신청 → 입금 확인 → 승인 → 결과`
- 왜 필요한가: 현재는 "대기 중"만 표시되어 불안감 유발

#### 매칭 신청 취소
- PENDING 상태에서 **신청 취소** 기능
- 취소 사유 선택 (실수로 신청, 금액 잘못 입금 등)
- 왜 필요한가: 실수로 신청한 경우 대처 방법이 없음 (withdraw API는 존재하지만 UI 미연결)

#### 온보딩 투어
- 첫 로그인 시 주요 기능 **가이드 툴팁** 표시
- "여기서 후보로 등록할 수 있어요" → "매칭을 신청해보세요" → "결과는 여기서 확인!"
- 왜 필요한가: 서비스 사용법을 직관적으로 전달

#### 다크 모드
- 축제는 주로 저녁에 활성화 → 야외에서 눈부심 방지
- 왜 필요한가: 축제 현장 사용성 향상

## 2. 관리자 관점

### 2-1. 대시보드 데이터

#### 실시간 통계 대시보드
현재 후보 수, 매칭 수 외에 추가로 필요한 데이터:

| 지표 | 설명 | 활용 |
|------|------|------|
| 시간대별 가입자 수 | 그래프 (막대/꺾은선) | 피크 타임 파악, 서버 리소스 계획 |
| 매칭 유형 비율 | RANDOM vs IDEAL 파이 차트 | 가격 정책 조정 근거 |
| 성별 비율 | 남/녀 후보자 수 비교 | 매칭 가능 인원 예측 |
| 평균 매칭 인원 수 | 1~5명 중 평균 | 수익 예측 |
| 결제 대기 건수 | 실시간 미처리 건 | 운영 우선순위 판단 |
| 일별 매출 | 날짜별 확정 결제 금액 합계 | 수익 추적 |
| 거절 사유 통계 | 사유별 빈도 | 반복 문제 파악 |
| 회원 전환율 | 가입 → 후보 등록 → 매칭 신청 퍼널 | 이탈 지점 파악 |

#### 매출 리포트
- 일별/주별 매출 요약 (총 결제액, 확정액, 대기액)
- CSV 다운로드 기능
- 왜 필요한가: 축제 종료 후 정산 및 보고서 작성

---

### 2-2. 결제 관리

#### 결제 목록 실시간 조회
- 현재 더미 데이터 사용 중 → **실제 결제 API 연동 필수**
- 결제 상태별 필터: 대기(WAITING) / 확정(CONFIRMED) / 실패(FAILED)
- 입금자명 + 금액 검색
- 왜 필요한가: 핵심 운영 기능이 아직 미구현

#### 일괄 승인/거절
- 체크박스로 여러 건 선택 → 한 번에 승인/거절
- 왜 필요한가: 축제 피크 시 건당 처리는 비효율적

#### 결제 알림
- 새 결제 대기 건 발생 시 **브라우저 알림** 또는 대시보드 뱃지
- "3건의 새 결제가 대기 중입니다"
- 왜 필요한가: 관리자가 대시보드를 계속 보고 있을 수 없음

---

### 2-3. 매칭 관리

#### 매칭 실행 인터페이스
- 관리자가 **매칭 실행 버튼**을 눌러 알고리즘 실행
- 실행 전 미리보기: "RANDOM 15건, IDEAL 8건 실행 예정"
- 실행 결과 요약: "23건 중 20건 성공, 3건 후보 부족으로 실패"
- 왜 필요한가: 매칭 실행 자체를 관리자가 트리거하는 UI가 없음

#### 매칭 결과 수동 조정
- 알고리즘 결과를 관리자가 검토/수정 후 확정
- 특정 매칭을 제거하거나 재배정
- 왜 필요한가: 알고리즘 오류나 특수 케이스 대응

#### 후보자 관리
- 후보자 목록 조회 (성별, MBTI, 등록일 필터)
- 후보자 강제 탈퇴/정지 기능
- 부적절한 프로필 내용 신고 처리
- 왜 필요한가: 운영 중 문제 후보자 관리 필요

---

### 2-4. 회원 관리

#### 회원 상세 조회 강화
- 특정 회원의 **전체 활동 이력** 조회 (가입 → 후보 등록 → 매칭 신청 → 결과)
- 결제 내역 연결
- 왜 필요한가: 문의 대응 시 회원 상황 빠르게 파악

#### 회원 검색 고도화
- 현재 기본 검색만 → **필터 조합 검색**
  - 성별, 역할(MEMBER/CANDIDATE/ADMIN), 가입일 범위, MBTI
- 왜 필요한가: 특정 조건의 회원 그룹 파악

#### 공지사항 관리 (CRUD)
- 관리자가 공지사항을 **작성/수정/삭제**
- 긴급 공지 상단 고정 기능
- 왜 필요한가: 현재 공지 데이터가 하드코딩됨

## 3. 공통 기능

### 3-1. 알림 시스템

| 알림 트리거 | 대상 | 내용 |
|-------------|------|------|
| 후보 등록 승인됨 | 회원 | "후보로 등록되었습니다!" |
| 매칭 신청 승인됨 | 회원 | "매칭 결과를 확인하세요!" |
| 신청 거절됨 | 회원 | "신청이 거절되었습니다. 사유: OOO" |
| 새 결제 대기 | 관리자 | "새 결제 확인 요청 N건" |
| 새 회원 가입 | 관리자 | "오늘 N명 가입" |

구현 방식:
- **1단계**: 인앱 알림 (벨 아이콘 + 알림 드롭다운)
- **2단계**: 브라우저 Push Notification (PWA)
- **3단계**: 이메일 알림 (선택적)

---

### 3-2. 신고/차단

- 매칭 결과에서 상대방 **신고** 버튼
- 신고 사유: 부적절한 프로필, 사칭, 불쾌한 연락 등
- 관리자 신고 처리 대시보드
- 왜 필요한가: 사용자 안전 보장, 축제 분위기 보호

---

### 3-3. 서비스 운영 타이머

- 축제 기간에만 서비스 활성화하는 **운영 시간 설정**
- 매칭 신청 마감 시간 카운트다운
- "D-2 매칭 마감까지 48시간 남았습니다"
- 왜 필요한가: 축제 한정 서비스의 긴급감 + 시간 관리

---

## 4. 우선순위 제안

### 즉시 필요 (축제 전 필수)
1. 결제 목록 실시간 조회 (관리자 — 현재 더미 데이터)
2. 매칭 실행 인터페이스 (관리자)
3. 실시간 통계 대시보드 확장 (관리자)
4. 진행 상태 실시간 안내 (사용자)
5. 공지사항 API 연동 (공통)

### 있으면 좋음 (축제 품질 향상)
6. 알림 시스템 — 인앱 알림 (공통)
7. 매칭 신청 취소 UI 연결 (사용자)
8. 일괄 승인/거절 (관리자)
9. 프로필 완성도 게이지 (사용자)
10. 온보딩 투어 (사용자)

### 장기적 개선 (다음 축제)
11. 매칭 만족도 피드백 (사용자)
12. MBTI 궁합 점수 (사용자)
13. 신고/차단 시스템 (공통)
14. 매출 리포트 + CSV (관리자)
15. 다크 모드 (사용자)
