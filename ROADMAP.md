# 기능 개발 순서

도메인 의존성 기준으로 정렬한다. 앞 단계가 완료되어야 다음 단계를 진행할 수 있다.

---

## Phase 0. 프로젝트 기반 구축

모든 기능의 전제 조건.

- [x] 라우팅 설정 (React Router — public / protected / admin 분리)
- [x] API 클라이언트 설정 (baseURL, 인터셉터, 토큰 자동 첨부, 401 갱신 처리)
- [x] 공통 컴포넌트 (Button, Input, Modal, Toast 등)
- [x] 레이아웃 (Header, Footer, 페이지 래퍼)
- [x] 전역 상태 관리 설정 (인증 상태 등)

---

## Phase 1. 인증

회원 기능 전체의 전제 조건.

- [x] 회원가입 (성별·MBTI·소개글·비밀번호 입력 + `@sangmyung.kr` 이메일 인증)
- [x] 로그인 / 로그아웃
- [x] Access Token / Refresh Token 관리 (자동 갱신) — axios 인터셉터 구조 구현
- [x] Protected Route 가드 (미인증 시 로그인 페이지로 리다이렉트)

---

## Phase 2. 메인 페이지

비회원도 접근 가능. 인증과 독립적이므로 Phase 1과 병행 가능.

- [x] 최근 매칭 소식 피드
  - `"OOO님이 매칭 후보로 등록되었습니다!"`
  - `"OOO님이 사랑을 찾기 위해 N명을 요청하였습니다!"`

---

## Phase 3. 회원 기능

Phase 1(인증) 완료 후 진행.

### 3-1. 프로필
- [x] 회원 정보 조회
- [x] 회원 정보 수정 (닉네임, MBTI, 소개글, 이상형)

### 3-2. 매칭 후보 등록
- [x] 후보 등록 신청 폼
- [x] 송금 안내 화면 (계좌/QR, 안내 사항 명시)
- [x] 등록 신청 내역 조회 (PENDING / APPROVED / REJECTED)

### 3-3. 매칭 신청
> 3-2 이후 진행 (후보가 있어야 매칭 가능)

- [x] 매칭 방식 선택 (무작위 / 이상형 기반)
- [x] 인원 선택 (1~5명) + 금액 안내 (RANDOM: 인당 1,000원 / IDEAL: 인당 1,500원)
- [x] 송금 안내 화면
- [x] 매칭 신청 내역 조회
  - 승인 대기 중 섹션 (결과 열람 불가)
  - 승인 완료 섹션 (결과 열람 가능)
  - 거절 섹션
- [x] 매칭 결과 열람 페이지

---

## Phase 4. 관리자 기능

Phase 3 완료 후 진행 (관리할 데이터가 있어야 함).

- [x] 회원 목록 조회 / 제한 처리
- [x] 후보 등록 신청 관리 (목록 조회 / 승인 / 거절)
- [x] 매칭 신청 관리 (목록 조회 / 승인 / 거절)
- [x] 결제 내역 관리 (확인 / 승인 / 거절)
- [x] Admin Route 가드 (`ROLE_ADMIN` 전용)
